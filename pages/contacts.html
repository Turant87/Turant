Селектор идентификатора

Существуют другие Selector, которые используются в особенных случаях. Селектор идентификатора (id) используют в интерактивных формах и для JavaScript. Есть селектор всех элементов на странице (*), его стоит использовать в редких случаях, например, изменение значения свойства box-sizing для удобства задания ширины всех элементов.

Можно составлять разные селекторы, указывая их отношения. Самое главное: браузер читает CSS справа налево. Рассмотрим отношение потомка:

  • селектор селектор — все потомки, пишется через пробел, header a выберет все ссылки (a) внутри шапки (header).
  • селектор > селектор — непосредственные потомки на один уровень вложенности, пишется через правую угловую скобку с пробелами вокруг знака. header > a — сработает так же. Использовать main > strong не получится, потому что strong еще вложен в p.
  • селектор + селектор — первый сосед. Выберет первый следующий элемент на том же уровне. h2 + p выберет первые параграфы, идущие после заголовков второго уровня.
